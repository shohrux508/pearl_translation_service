# Pearl Translation Service 🤖📄

Асинхронный Telegram-бот и Backend-сервис для умного перевода и оцифровки личных документов с использованием ИИ **Google Gemini**. Бот принимает фотографию документа, извлекает и переводит данные, позволяет пользователю внести правки, а затем возвращает готовый переведенный документ в формате Word (`.docx`).

## ✨ Основные возможности

- **Многостраничное распознавание**: Поддержка загрузки нескольких фотографий (например, обе стороны ID-карты) для более точного извлечения данных.
- **Умное распознавание (AI)**: Извлечение текста и данных из фотографий документов с помощью **Gemini 2.5 Flash**.
- **Генерация полей с помощью ИИ**: При добавлении нового документа администратором ИИ автоматически подбирает ключи и переводит названия полей на английский язык, минимизируя ручной ввод.
- **Интерактивная валидация**: После распознавания бот предоставляет удобное меню (inline-кнопки), где пользователь может проверить и точечно отредактировать любое поле перед генерацией итогового файла.
- **Генерация .docx**: Автоматическая подстановка извлеченных данных в заранее подготовленные Word-шаблоны посредством `docxtpl`.
- **Мультиязычность**: Поддержка перевода на русский и английский языки.
- **Динамическое добавление документов**: Администраторы могут добавлять новые типы документов и шаблоны прямо через Telegram (команда `/add_doc`), без необходимости изменять исходный код. 
- **API (FastAPI)**: Наличие REST API сервиса, позволяющего интегрировать функционал в другие приложения.
- **Гибкая конфигурация**: Настройка полей, промптов и типов документов хранится в `documents.json`.
- **Легкий деплой**: Полная готовность к развертыванию на платформах типа **Railway**.

## 🛠 Стек технологий

- **Язык**: Python 3.12+
- **Telegram Bot API**: [aiogram 3.x](https://docs.aiogram.dev/)
- **Нейросети**: [google-generativeai](https://pypi.org/project/google-generativeai/) (Gemini API)
- **Web-фреймворк**: [FastAPI](https://fastapi.tiangolo.com/) (для API и управления сервером) + Uvicorn
- **Работа с документами**: [docxtpl](https://docxtpl.readthedocs.io/) (Jinja2 для Word)
- **Архитектура**: Разделение на слои, легкий Dependency Injection контейнер.
- **Тестирование**: `pytest`

## 🚀 Установка и запуск

1. **Клонируйте репозиторий**:
   ```bash
   git clone <url-вашего-репозитория>
   cd pearl_translation_service
   ```

2. **Создайте виртуальное окружение и установите зависимости**:
   ```bash
   python -m venv .venv
   # Для Windows:
   .venv\Scripts\activate
   # Для Linux/macOS:
   source .venv/bin/activate
   
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения**:
   Создайте файл `.env` в корневой папке на основе доступных параметров:
   ```env
   BOT_TOKEN=ваш_токен_telegram_бота
   GEMINI_API_KEY=ваш_ключ_от_google_gemini
   API_HOST=0.0.0.0
   PORT=8000
   RUN_TELEGRAM=True
   RUN_API=True
   ```

4. **Запустите проект**:
   ```bash
   python main.py
   ```

## 📝 Использование бота

### Для обычных пользователей
1. Отправьте боту первую фотографию документа (как фото). При необходимости бот предложит отправить и оборотную сторону.
2. Выберите тип документа из предложенного меню (ID карта, Паспорт и др.).
3. Выберите язык перевода (Русский / Английский).
4. Ожидайте, пока Gemini проанализирует фото и извлечет данные.
5. Проверьте результат. Если есть ошибки распознавания, нажмите на соответствующее поле (inline-кнопку) и отправьте исправленный текст.
6. Нажмите **"Подтвердить и создать"**, после чего бот пришлет готовый `.docx` файл.

### Для администраторов
1. Отправьте команду `/add_doc` в чате с ботом.
2. Следуйте пошаговым инструкциям бота:
   - Введите уникальный ID документа (`marriage_cert`).
   - Название (`Свидетельство о браке`).
   - Эмодзи (💍).
   - Список названий полей на русском языке (каждое с новой строки). *Бот автоматически сгенерирует ключи и английские переводы списка полей с помощью ИИ.*
   - Загрузите `.docx` шаблоны для русского и английского языков (они сохранятся в папку `templates/`).
3. Новый тип документа сразу станет доступен для выбора всем пользователям, а данные автоматически запишутся в `documents.json`.

## 🧪 Тестирование

Проект использует фреймворк **pytest** для автоматизированного тестирования. 

Для запуска тестов выполните команду:
```bash
pytest
```
*В тестах реализованы проверки основных сценариев, мокирования ответов от Gemini API, а также валидация конфигурации документов.*

## ⚙️ Конфигурация документов (`documents.json`)

Вся информация о полях для извлечения, промптах для AI и локализации хранится в `documents.json`. 
Пример структуры:
```json
{
    "document_types": {
        "id_card": { "name": "ID карта", "emoji": "🪪" }
    },
    "configs": {
        "id_card": { "prompt_fields": "given_name, surname, citizenship..." }
    },
    "field_names": {
        "ru": { "given_name": "Имя", "surname": "Фамилия" },
        "en": { "given_name": "Given Name", "surname": "Surname" }
    }
}
```
Вы можете редактировать этот файл вручную или использовать команду `/add_doc` прямо через Telegram.
