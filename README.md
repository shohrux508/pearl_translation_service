**Pearl Translation Service** — это современный многофункциональный Telegram-бот и Backend-сервис для умного перевода и оцифровки личных документов с использованием ИИ **Google Gemini**. В основе архитектуры лежит связка `aiogram 3` для взаимодействия с пользователем и `FastAPI` (в перспективе REST API) для управления серверной частью, работающие внутри единого асинхронного event loop.

Бот принимает фотографию (или несколько фотографий) документа, интеллектуально извлекает и переводит данные в структурированном виде, позволяет пользователю легко внести правки через inline-меню, а затем возвращает готовый переведенный документ в формате Word (`.docx`), сгенерированный по заранее заготовленному шаблону.

---

## ✨ Основные возможности (Подробно)

1. **Многостраничное распознавание**
   Бот поддерживает состояние ожидания (FSM) для загрузки нескольких фотографий одновременно (например, обе стороны ID-карты или многостраничный паспорт). Это позволяет ИИ комбинировать данные с разных страниц для более полного и точного заполнения полей.

2. **Умное извлечение (AI - Gemini 2.5 Flash)**
   Для анализа изображений используется multimodal модель Google Gemini. ИИ не просто считывает текст (как классический OCR), а *понимает* контекст: находит нужные поля (имя, фамилия, номера лицензий), переводит их и форматирует ответ в строгий JSON формат, игнорируя лишнюю визуальную информацию.

3. **Генерация полей с помощью ИИ (Для администраторов)**
   При добавлении нового типа документа прямо из Telegram, администратор вводит только русские названия полей. Бот автоматически отправляет их в Gemini для подбора оптимальных английских ключей (keys) и перевода названий, навсегда избавляя от рутинной настройки `json` конфигураций.

4. **Интерактивная валидация данных**
   После успешного извлечения данных пользователю не отправляется сразу готовый файл. Бот предоставляет удобное меню с помощью inline-кнопок, где видно текущее распознанное значение для каждого поля. Если модель ошиблась из-за блика на фото или сложной печати, пользователь может нажать на поле и прислать правильный текст.

5. **Генерация `.docx` файлов на лету**
   Автоматическая подстановка финально утвержденных данных в готовые Word-шаблоны посредством библиотеки `docxtpl` (на базе синтаксиса Jinja2: `{{ var_name }}`). Бот мгновенно генерирует файл и отправляет его обратно в чат.

6. **Встроенная мультиязычность**
   Поддержка перевода на различные языки. В конфигурации жестко закреплена поддержка шаблонов для различных языков (по умолчанию реализованы Русский и Английский шаблоны перевода).

7. **Динамическое добавление документов (No-Code подход)**
   Удобный административный интерфейс: вам не нужен Python-разработчик, чтобы добавить новый тип документа. Вы можете добавить новые типы документов и шаблоны прямо через Telegram с помощью удобной команды `/add_doc`.

8. **Слоистая архитектура и Dependency Injection (DI)**
   Проект написан с использованием чистого кода. Используется легковесный контейнер для прокидывания сервисов (`GeminiTranslationService`, конфиги) в хендлеры и FastAPI-роутеры без создания жесткого связывания. Внедрены принципы DRY и SOLID.

## 🛠 Стек технологий

- **Язык программирования:** Python 3.12+
- **Telegram интерфейс:** [aiogram 3.x](https://docs.aiogram.dev/) — современный асинхронный фреймворк для Telegram.
- **Нейросети:** [google-generativeai](https://pypi.org/project/google-generativeai/) — официальная библиотека API Google Gemini.
- **Веб-сервер / API:** [FastAPI](https://fastapi.tiangolo.com/) + Uvicorn — для обеспечения деплоймента, REST API и Cloud Healthcheck.
- **Шаблонизация документов:** [docxtpl](https://docxtpl.readthedocs.io/) — генерация Word-документов.
- **Тестирование:** `pytest` — фреймворк для юнит-тестирования.
- **Вспомогательные утилиты:**
  - `pydantic-settings` — надежное управление конфигурацией.
  - `python-dotenv` — локальные переменные среды окружения.
  - `Pillow` — работа с изображениями.

## 📂 Структура проекта

Кодовая база разделена на логические слои для обеспечения легкого масштабирования:

```text
pearl_translation_service/
├── app/
│   ├── api/               # FastAPI роутеры (REST API для внешних интеграций)
│   ├── telegram/
│   │   ├── routers/       # Роутеры aiogram:
│   │   │   ├── translator.py   # Основной флоу обычного пользователя (перевод)
│   │   │   └── admin_docs.py   # Сценарии администратора (добавление шаблонов)
│   │   └── bot.py         # Инициализация Dispatcher и Bot
│   ├── services/
│   │   ├── gemini_service.py    # Класс интеграции с Google Gemini AI
│   │   └── document_manager.py  # Управление конфигурацией types/fields
│   ├── app.py             # Оркестратор: старт FastAPI и Bot в одном loop
│   ├── config.py          # Валидированные настройки окружения
│   └── container.py       # Легковесный DI-контейнер
├── templates/             # Главное хранилище .docx шаблонов ({{ field_name }})
├── temp/                  # Временная папка для загруженных фото и сгенерированных файлов
├── translate_id.py        # Утилитарный/тестовый скрипт для работы с docxtpl
├── main.py                # Entrypoint запуска бизнес-логики
├── documents.json         # Хранилище типов документов и локализаций (in-file БД)
├── .env                   # Файл с секретными ключами (не отслеживается в git)
└── requirements.txt       # Индекс Python зависимостей
```

## 🚀 Детальная установка и запуск

### 1. Подготовка системы
Убедитесь, что на вашей машине установлен **Python версии 3.12** или более свежей.

```bash
# Клонирование репозитория
git clone <url-вашего-репозитория>
cd pearl_translation_service
```

### 2. Настройка виртуального окружения
Всегда используйте виртуальное окружение, чтобы избежать конфликтов пакетов в системе.

```bash
# Создание виртуального окружения
python -m venv .venv

# Активация окружения (для Windows):
.venv\Scripts\activate

# Активация окружения (для Linux/macOS):
source .venv/bin/activate
```

Установите требуемые зависимости:
```bash
pip install -r requirements.txt
```

### 3. Настройка переменных среды `.env`
Создайте в корневой директории проекта конфигурационный файл `.env` и добавьте в него следующие настройки:

```env
# Обязательные ключи аутентификации
BOT_TOKEN=1234567890:ААBВСС...          # Токен от @BotFather
GEMINI_API_KEY=AIzaSy...                 # Developer API Ключ из Google AI Studio

# Настройки сетевого интерфейса веб-сервера
API_HOST=0.0.0.0
PORT=8000

# Переключатели модулей системы (Feature flags)
RUN_TELEGRAM=True                        # Включить модуль Telegram-бота
RUN_API=True                             # Включить модуль FastAPI Web-сервера
```

### 4. Запуск приложения
Запустите основной входной файл через интерпретатор python. Приложение распараллелит запуск Long Polling для Telegram и веб-сервер uvicorn в асинхронном режиме.

```bash
python main.py
```
В консоли вы увидите логи инициализации сервисов и уведомление: `Application stopped` при выключении.

## 📝 Руководство пользователя

### 👤 Основной Workflow (Пользователь)
1. Отправьте боту команду `/start`.
2. Бот предложит загрузить документ в виде фотографии (или медиагруппы, если документ двусторонний). 
3. На экране появится меню со списком известных боту типов документов (например, `ID карта`, `Свидетельство о рождении`, `Паспорт`).
4. Выберите язык целевого переведенного документа (например, **Английский**).
5. Начнется "магия": AI отправляет фотографию и системный промпт в Google Gemini.
6. **Валидация:** Бот присылает сообщение со списком всех распознанных полей под фотографией (каждое поле — это inline-кнопка).
   - Если всё идеально — смело нажимайте зеленую кнопку **"✅ Подтвердить и создать"**.
   - Если в поле `Фамилия` допущена опечатка (AI не так понял шрифт) — кликайте на кнопку `Фамилия`. Бот попросит ввести верный текст. Отправка исправления мгновенно обновит черновик.
7. В финале бот присылает вам готовый Word-файл (`.docx`), заполненный вашими данными.

### 👑 Workflow для Администраторов (Кастомные шаблоны)
Эта функция делает проект максимально гибким и избавляет администраторов от работы с кодом.

1. Отправьте боту команду `/add_doc` (доступна настроенным администраторам).
2. Следуйте пошаговому мастеру настройки:
   - Введите уникальный системный **ID** (на латинице без пробелов, например: `driver_license`).
   - Введите публичное **название** документа (`Водительские права`).
   - Назначьте релевантный **Эмодзи** (`�`).
   - Отправьте боту **список названий полей на русском языке**, каждое поле с новой строки. Например:
     ```text
     Фамилия
     Имя
     Дата рождения
     Категория прав
     ```
   - **AI Ассистирование:** Модель Gemini автоматически распознает ваш ручной ввод, придумает системные уникальные ключи (т.е., `surname`, `given_name`, `date_of_birth`, `categories`) и переведет названия на английский язык.
   - Загрузите `.docx` файлы-шаблоны для Русского и Английского перевода. Важно использовать теги Jinja внутри документа, например `{{ surname }}`, `{{ given_name }}`. Бот самостоятельно сохранит их в директорию `templates/`.
3. Новый документ появится в меню выбора у всех пользователей!

## ⚙️ Устройство конфигурации `documents.json`

Файл `documents.json` заменяет собой статичную системную конфигурацию (State Storage). 
Структура файла включает 3 основных JSON-вложенности:

- `document_types` — корневой реестр доступных видов документов и их метаданные (name, emoji).
- `configs` — связь между документом и списком полей (словарь keys), которые ищет ИИ на снимках.
- `field_names` — словари локализации (как `surname` будет отображаться на русском ("Фамилия") или на английском ("Surname") языке в клавиатурах).

Фрагмент структуры `documents.json`:
```json
{
    "document_types": {
        "id_card": { "name": "ID карта", "emoji": "🪪" }
    },
    "configs": {
        "id_card": { 
           "prompt_fields": "given_name, surname, date_of_birth, serial_code..." 
        }
    },
    "field_names": {
        "ru": { 
           "given_name": "Имя", 
           "surname": "Фамилия" 
        },
        "en": { 
           "given_name": "Given Name", 
           "surname": "Surname" 
        }
    }
}
```

## 🧪 Тестирование (Pytest)

Процесс тестирования построен на базе фреймворка `pytest` (в соответствии с глобальными правилами пользователя).

Покрытие тестами (`app/tests` или встроенные модули):
- Валидация загрузки и структуры конфигурации `documents.json` (проверка синтаксиса и обязательных JSON-нод).
- Мокирование асинхронных ответов `google-generativeai`. Это гарантирует предсказуемость ответов и успешное тестирование парсинга ответов бота без реального (платного) вызова API во время CI/CD процессов.
- Валидация чистоты архитектуры и работы Dependency Injection.

Запуск набора тестов в корне репозитория:
```bash
pytest
# или же с глубоким детальным логированием
pytest -v -s
```
