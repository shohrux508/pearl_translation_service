![Python](https://img.shields.io/badge/Python-3.12+-blue.svg)

**Pearl Translation Service** — это современный многофункциональный Telegram-бот и Backend-сервис для умного перевода и оцифровки личных и бизнес-документов с использованием ИИ **Google Gemini**. В основе архитектуры лежит связка `aiogram 3` для взаимодействия с пользователем и `FastAPI` (в перспективе REST API) для управления серверной частью, работающие внутри единого асинхронного event loop.

Бот принимает фотографии документов (поддерживается многостраничность), интеллектуально извлекает и переводит данные строго по сгенерированной **JSON Schema**, позволяет пользователю вносить правки через продвинутое inline-меню (с поддержкой таблиц и вложенностей), а затем возвращает готовый переведенный документ в формате Word (`.docx`), сгенерированный по заранее заготовленному шаблону.

---

## ✨ Основные возможности (Подробно)

1. **Многостраничное распознавание и агрегация**
   Бот поддерживает состояние ожидания (FSM) для загрузки **нескольких фотографий одновременно** (например, обе стороны ID-карты, многостраничный паспорт или договор). Бот не начинает анализ до нажатия кнопки «Начать распознавание», что позволяет ИИ собрать весь контекст со всех страниц и объединить данные для максимально точного заполнения полей.

2. **Строгое извлечение данных (Strict JSON Schema + Gemini 2.5 Flash)**
   Для анализа изображений используется multimodal модель Google Gemini, запрос к которой происходит через **Strict JSON Schema**. ИИ не просто считывает текст (как классический OCR), а *понимает* контекст: находит нужные поля (имя, фамилия, таблицы из инвойсов), переводит их и возвращает ответ в **строго валидированном JSON формате**, полностью исключая галлюцинации ключей или потерю данных.

3. **Сложные структуры данных: Таблицы и Вложенность**
   Теперь бот поддерживает не просто плоские ключи, но и сложную структуру документа. Вы можете распознавать целые **табличные данные** (например, списки товаров с ценами) и вложенные свойства. Для Telegram-интерфейса реализовано удобное Summary-отображение таблиц с пагинацией и интерфейсом построчного редактирования.

4. **Продвинутая интерактивная валидация данных (Богатый UI)**
   После успешного извлечения данных пользователю не отправляется сразу готовый файл. Бот предоставляет удобное inline-меню со следующими возможностями:
   - **Поэкранная проверка:** Каждое поле отображается как кнопка. Нажали на кнопку — ввели исправленный вариант текстом.
   - **Табличный UI:** Просмотр и редактирование отдельных строк сложных таблиц прямо из чата.
   - **Режим Raw JSON:** Для сложных случаев опытный пользователь может нажать «Редактировать всё (JSON)» и прислать боту сырой исправленный JSON объект для моментального обновления всего документа целиком.

5. **Умные инструкции (Smart UX)**
   При загрузке бот предлагает советы по правильной съемке документов (освещение, фокус, отсутствие бликов), снижая вероятность ошибок со стороны AI и улучшая пользовательский опыт (Smart Photo Instructions).

6. **Генерация полей ИИ и переводы (Для администраторов)**
   При добавлении нового типа документа прямо из Telegram, администратор вводит только русские названия полей. Бот автоматически отправляет их в Gemini для подбора оптимальных английских ключей (keys), обеспечивая No-Code подход для расширения системы.

7. **Мгновенная генерация `.docx`**
   Автоматическая подстановка финально утвержденных данных (и даже целых таблиц) в готовые Word-шаблоны посредством библиотеки `docxtpl` (на базе синтаксиса Jinja2: `{{ var_name }}`, `{% for row in table %}`). Бот мгновенно генерирует файл и отправляет его обратно в чат.

8. **Слоистая архитектура и Dependency Injection (DI)**
   Проект написан с использованием чистого кода. Используется легковесный контейнер для прокидывания сервисов (`GeminiTranslationService`, `DocumentManager`) в хендлеры и FastAPI-роутеры без жесткого связывания.

## 🛠 Стек технологий

- **Язык программирования:** Python 3.12+
- **Telegram интерфейс:** [aiogram 3.x](https://docs.aiogram.dev/) — современный асинхронный фреймворк для Telegram.
- **Нейросети:** API Google Gemini (через `google-generativeai`) с функцией Structured Outputs (JSON Schema).
- **Веб-сервер / API:** [FastAPI](https://fastapi.tiangolo.com/) + Uvicorn — REST API и Cloud Healthcheck.
- **Шаблонизация документов:** [docxtpl](https://docxtpl.readthedocs.io/) — мощная генерация Word-документов с поддержкой циклов и условий.
- **Тестирование:** `pytest` — фреймворк для юнит-тестирования (мокирование API, проверка стейтов).
- **Вспомогательные утилиты:** `pydantic`, `pydantic-settings`, `python-dotenv`, `Pillow`.

## 📂 Структура проекта

Кодовая база разделена на логические слои для обеспечения легкого масштабирования:

```text
pearl_translation_service/
├── app/
│   ├── api/               # FastAPI роутеры (REST API для внешних интеграций)
│   ├── telegram/
│   │   ├── routers/       # Роутеры aiogram:
│   │   │   ├── translator.py   # Основной флоу пользователя (фото, валидация, таблицы, FSM)
│   │   │   ├── admin_docs.py   # Сценарии администратора (добавление шаблонов с AI)
│   │   │   └── admin_manage.py # Управление ботом
│   │   └── bot.py         # Инициализация Dispatcher и Bot
│   ├── services/
│   │   ├── gemini_service.py    # Провайдер Gemini AI (теперь с поддержкой Strict Schema)
│   │   ├── document_manager.py  # Управление конфигурацией types/fields и генерация Schema
│   │   └── docx_service.py      # Рендеринг docxtpl
│   ├── app.py             # Оркестратор: старт FastAPI и Bot в одном loop
│   ├── config.py          # Валидированные настройки окружения
│   └── container.py       # Легковесный DI-контейнер
├── templates/             # Главное хранилище .docx шаблонов ({{ field_name }})
├── temp/                  # Временная папка для загруженных фото и сгенерированных файлов
├── main.py                # Entrypoint запуска бизнес-логики
├── documents.json         # Хранилище типов документов, схем данных и локализаций
├── .env                   # Секретные ключи
└── requirements.txt       # Индекс зависимостей Python
```

## 🚀 Детальная установка и запуск

### 1. Подготовка системы
Убедитесь, что установлен **Python 3.12** или старше.

```bash
git clone <url-вашего-репозитория>
cd pearl_translation_service
```

### 2. Настройка виртуального окружения

```bash
python -m venv .venv

# Для Windows:
.venv\Scripts\activate
# Для Linux/macOS:
source .venv/bin/activate

pip install -r requirements.txt
```

### 3. Настройка окружения `.env`
Создайте `.env` и добавьте ключи:

```env
# Аутентификация
BOT_TOKEN=1234567890:ААBВСС...          # Токен от @BotFather
GEMINI_API_KEY=AIzaSy...                 # Developer API Ключ из Google AI Studio

# Настройки сети
API_HOST=127.0.0.1
PORT=8000

# Feature flags
RUN_TELEGRAM=True
RUN_API=True
```

### 4. Запуск приложения

```bash
python main.py
```

## 📝 Руководство пользователя

### 👤 Основной Workflow (Пользователь)
1. Отправьте `/start`, бот предложит удобное меню навигации.
2. При нажатии "Перевести документ" ознакомьтесь с быстрыми инструкциями по фото (для лучшего качества).
3. Загрузите **одно или несколько фото** документа по одной или сгруппировав их. Нажмите *"✅ Начать распознавание"*.
4. Выберите тип документа (Паспорт, Свидетельство, Инвойс и т.д.) и язык перевода (Английский, Русский и др.).
5. AI проводит обработку с использованием жесткой схемы:
6. **Валидация:**
   - Вы получите список распознанных данных. Простые поля можно радактировать прямо из кнопок.
   - Если документ содержит таблицы (Table View), вы можете зайти внутрь "Посмотреть таблицу", перелистывать страницы, редактировать отдельные строки данных.
   - Опытным пользователям доступна кнопка **"⚙️ Редактировать сырой JSON"**.
7. Утвердите данные ("✅ Подтвердить и создать"), и бот сгенерирует финальный Word-файл (`.docx`).

### 👑 Workflow для Администраторов (Кастомные шаблоны)
1. Отправьте команду `/add_doc` для запуска мастера.
2. Укажите ID (например, `rental_contract`), название и Эмодзи.
3. Отправьте список названий полей на русском. AI всё сам переведет и создаст ключи.
   *(Для вложенных или табличных схем может потребоваться ручная корректировка `documents.json` под нужную иерархию).*
4. Загрузите .docx шаблоны, используя сгенерированные jinja-ключи для нужных локализаций.
5. Готово!

## ⚙️ Устройство конфигурации `documents.json` (Схемы данных)

Файл `documents.json` это ядро типизации системы. Структура:
- `document_types` — реестр документов.
- `configs` — содержит `prompt_fields` как для плоских списков, так и для описания сложных табличных узлов или словарей `dict`. Служит основой для динамической генерации pydantic/JSON Schema перед отправкой в Gemini.
- `field_names` — словари локализации.

_Благодаря строгому парсингу, ключи из `configs` гарантированно возвращаются из Gemini, что обеспечивает отсутствие ошибок при рендеринге `docxtpl`._
